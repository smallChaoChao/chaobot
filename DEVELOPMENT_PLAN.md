# 🤖 chaobot 开发计划

## 项目概述

chaobot 是一个轻量级个人 AI 助手，灵感来源于 nanobot 和 OpenClaw，采用现代化架构设计，集成了两者的优点并解决了潜在问题。

---

## 📊 竞品分析

### nanobot 核心特性

| 特性 | 描述 | chaobot 状态 |
|------|------|-------------|
| **消息总线** | `asyncio.Queue` 异步消息队列，解耦频道和 Agent | ✅ 已实现 |
| **多频道支持** | 飞书、Telegram、Discord 等 | 🚧 部分实现 (Feishu) |
| **子代理系统** | 后台任务执行，处理长时间运行任务 | 📋 待开发 |
| **双层记忆** | MEMORY.md (长期) + HISTORY.md (可搜索历史) | 📋 待开发 |
| **MCP 工具** | Model Context Protocol 动态工具加载 | 📋 待开发 |
| **Skill 系统** | Markdown 文档扩展 AI 能力 | ✅ 已实现 |
| **定时任务** | Cron 表达式调度 | 📋 待开发 |
| **心跳监控** | 系统健康状态监控 | 📋 待开发 |

### OpenClaw 核心特性

| 特性 | 描述 | chaobot 状态 |
|------|------|-------------|
| **本地优先部署** | 完全本地运行，数据隐私保护 | 📋 待开发 |
| **浏览器自动化** | 网页浏览、点击、表单填写、截图 | 📋 待开发 |
| **定时任务执行** | 定时收集信息、自动执行工作流 | 📋 待开发 |
| **长期记忆** | 持久化记忆，跨会话保持上下文 | 📋 待开发 |
| **多代理协同** | 多个 Agent 协同完成复杂任务 | 📋 待开发 |
| **Skills 市场** | 3000+ 技能插件，扩展能力强 | 🚧 部分实现 |
| **邮件/日历管理** | 自动清理收件箱、管理日程 | 📋 待开发 |
| **主动执行** | 无需指令主动完成任务 | 📋 待开发 |
| **多平台集成** | 飞书、QQ、Discord 等 | 🚧 部分实现 |

### chaobot 当前优势

| 特性 | 描述 |
|------|------|
| **配置管理** | Pydantic 验证 + 环境变量支持 |
| **类型安全** | 完整的 Python 类型注解 |
| **全异步架构** | 所有 I/O 操作使用 asyncio |
| **工具注册表** | 插件化架构，易于扩展 |
| **Provider 规范** | 统一的 LLM Provider 接口 |
| **Dashboard UI** | Web 可视化配置管理 |
| **Session 管理** | Web UI 管理对话会话 |
| **UI/UX Skill** | 内置设计系统生成能力 |

## 📝 设计原则

1. **保持简洁**: 参考 nanobot 的轻量级理念，避免过度工程化
2. **类型安全**: 使用 Python 3.10+ 的类型注解
3. **异步优先**: 所有 I/O 操作使用 asyncio
4. **配置驱动**: 通过配置文件控制功能，减少硬编码
5. **向后兼容**: 版本升级时保持配置兼容
6. **模块化**: 每个功能独立模块，清晰接口
7. **可扩展**: 插件化架构，易于添加新功能
